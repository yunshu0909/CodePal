## 记忆协议（详细）

### 冷启动

首次读取 `MEMORY.md` 若为空模板（只有标题和示例注释）：

1. 主动问用户：你的角色？主要用 AI 做什么？对话风格偏好？有无特别工作习惯？
2. 扫描目录 / README / 配置，生成 2-3 句项目简介，与用户确认。
3. 把信息填进 `MEMORY.md` 对应小节，并更新 `AGENTS.md` / `CLAUDE.md` 的「项目简介」。
4. 完成后正常进入记忆工作流。

### 长期记忆 `MEMORY.md`

- 稳定核心信息：个人画像、偏好、关键决策、项目状态。每次对话自动加载。
- 超约 200 行优先拆到 `memory/topics/xxx.md` 按需加载。
- 蒸馏时**改写**对应条目（新信息与旧记录矛盾就覆盖），而非无限追加。

### 每日记忆 `memory/YYYY-MM-DD.md`

- 当天结论 / 洞察 / 待办 / 决策。
- 对话开始读**最近 7 天**；更早历史搜 `memory/`。
- 对话结束自主判断写入，不必每次问；用户说「记一下」立刻写。
- 积累超 7 天主动提议蒸馏，把仍有价值的并入 `MEMORY.md`。

### 注意

- 记忆以项目根目录文件为准；**忽略 `~/.claude/` 的 auto-memory**，不读不写。
