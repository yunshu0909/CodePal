## 记忆协议

### 冷启动规则

首次读取 MEMORY.md 时，如果内容为空模板（只有标题框架没有实际内容）：

1. **主动引导用户完成初始化**，依次询问：
   - 你的角色是什么？（如：开发、产品、运营、SA）
   - 你主要用 AI 做什么？（如：写代码、技术方案、文档、数据分析）
   - 对话风格偏好？（简洁直接 / 详细展开 / 随意聊天）
   - 有什么特别的工作习惯需要了解？（可选，用户可跳过）
2. **自动生成项目简介**：扫描目录结构、README、配置文件等，生成 2-3 句概述，让用户确认或修改
3. **填充 MEMORY.md**：将收集到的信息写入对应区域（关于我、项目与上下文、工作习惯）
4. **更新本文件的「项目简介」章节**
5. 完成后正常进入记忆协议工作流

### 长期记忆（MEMORY.md）

- 存放稳定的核心信息：个人画像、工作偏好、关键决策
- 每次对话自动加载，始终在上下文中
- 超过 150 行时主动提议精简

### 每日记忆（memory/YYYY-MM-DD.md）

- 存放当天对话产生的要点：结论、洞察、待办、决策
- 对话结束时自主判断是否有值得记录的内容，有就直接写入，不需要每次问用户确认
- 需要回顾历史时，搜索 memory/ 目录检索

### 工作流

1. 对话开始 → 读取 MEMORY.md + memory/当天日期.md + memory/昨天日期.md，了解上下文
2. 涉及历史上下文 → 搜索 memory/ 目录
3. 对话结束 → 自主判断有无值得记录的内容，有就写入 memory/当天日期.md
4. 产出长期价值信息（如重要决策、新发现）→ 同步更新 MEMORY.md
5. 注意：所有路径相对于项目根目录，不是全局 ~/.claude/ 目录

### 补充规则

- 记忆以项目根目录下的文件为准，忽略 ~/.claude/ 下的 auto-memory（内置记忆系统），不要读也不要写
- 用户说"记住这个"、"这个记一下"时，立刻写入文件，不要只"记在心里"等结束再存
- 每日记忆积累超过 7 天时，主动提议做一次蒸馏：把仍然有价值的要点合并到 MEMORY.md，过时的标记或清理
